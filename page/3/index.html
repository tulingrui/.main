简单的nginx搭建负载均衡与根据URL关键字转发服务

post @ 2018-12-06

简单的nginx搭建负载均衡与根据URL关键字转发服务

项目要做前后端分离

后端服务要做负载均衡

本地先试1下 start

首先

写两个集群 

upstream hd_cloud {  #后端服务集群名字   
    serverlocalhost:9091  weight=1;
    serverlocalhost:9092  weight=1;
    serverlocalhost:9093  weight=1;
    serverlocalhost:9094  weight=1;
}

upstream hd_client{  #前端服务集群名字   
    serverlocalhost:8081;
}

weight是权重的意思，权重越大，分配的概率越大。

然后是 配置server

listen 80;
server_name  localhost;
location ~/action {
    proxy_pass http://hd_cloud;  #后端集群的名 上面写的 upstream hd_cloud
    proxy_connect_timeout 1; #连接超时时间
    index  index.html index.htm;
}

location / {
    proxy_pass http://hd_client; #前端集群的名 
    proxy_connect_timeout 1; #连接超时时间
    index  index.html index.htm;
}

location ~/action 匹配路径如果访问路径中包含action 就访问后端服务

location / 表示通用匹配

“~” 开头表示区分大小写的正则匹配

“~*” 开头表示不区分大小写的正则匹配

“/“ 通用匹配，任何请求都会匹配到

还有其他的规则不列举了

nginx的匹配顺序是 先匹配正则 再匹配普通

服务启动 试1下 可以了。

nginx的关闭与重新载入配置信息

退出
nginx.exe -s quit  

重新载入配置信息
nginx.exe -s reload

写个bat文件放在nginx目录中方便一些

Jenkins+Maven+SVN+Tomcat搭建自动部署工具

项目更新比较频繁，每次都要上服务器 复制 粘贴 重启 很烦

百度了1下 Jenkins/Maven/SVN/Tomcat 嗯 开搞8

首先准备好 Maven、SVN、Tomcat（还有环境变量什么的）

Jenkins 下载地址 ↓

https://jenkins.io/download/

我是放Tomcat下，下的是war包 (Generic Java package)

下载好把它复制到 tomcat 的 webapps 文件夹中

启动1下 Tomcat ，可以访问了 http:/localhost:8080/jenkins/

Jenkins初始化页面会提示你输入密码， 按提示路径打开密码文件，输入密码确定

记点重要的

先进入插件管理安装必需的插件

  1. Build Timeout
  2. Deploy to container Plugin
  3. Email Extension Plugin
  4. Gradle Plugin
  5. LDAP Plugin
  6. Matrix Authorization Strategy Plugin
  7. OWASP Markup Formatter Plugin
  8. Pipeline
  9. Pipeline Maven Integration Plugin
  10. .Publish Over SSH
  11. SSH plugin
  12. SSH Slaves plugin
  13. Subversion Plug-in
  14. Timestamper
  15. Workspace Cleanup Plugin

安装完重启1下Tomcat

接下来 系统设置

设置编码全局属性

Name : LANG

Value : zh_CN UTF-8

增加系统管理员邮件地址

Jenkins Location ：

Jenkins URL - Jenkins的访问地址 http://localhost:8080/jenkins

System Admin e-mail address youEmail@qq.com

其他暂时先默认的

然后是 添加全局工具配置 Global ToolConfiguration

将JDK取消自动安装 把本机的JDK路径填上(例：E:\Java\jdk1.8.0_171)

Maven也一样取消自动安装，将本机的Maven路径填上(例：D:\apache-maven-3.5.4)

接着 可以开始一个任务了 - New任务

构建一个自由风格的软件项目或者根据需要选类型

新建完在左侧可以看到1些按钮，接下来可以配置（Configure）项目了

General配置

Description - 项目介绍

Days to keep builds - 保持构造的天数 （我设置的 30）

Max # of builds to keep -保持构造的最大个数 （100）

然后是 代码管理(Source Code Management) 也就是 设置你项目的SVN地址

Repository URL 输入你SVN的地址

点添加 输你SVN的账号密码

还可以配置构建触发器（Build Triggers），但后来用不到就没配置了

然后是构造 （Build Environment）

勾选 构造之前删除工作空间 (Delete workspace before build starts)

Maven Version ：maven-3.5.4

Goals ：clean install

POM :pom.xml

接着是构造后的操作 (Post-build Actions)

WAR/EAR files - 指的是你构造工作空间的目录

我的是 C:\Users\DELL.jenkins\workspace\target\manageTool.war

那么 填 target/manageTool.war 就可以了

Context path - 是配置项目访问路径

如填 /manageTool 项目根访问目录就是：http://localhost:8080/manageTool

Add Container 选你对应的Tomcat版本， add 添加你Tomcat的账号密码与Tomcat地址

Tomcat的用户配置 在 conf 下的 tomcat-users.xml 文件中

例：

<tomcat-users>
    <role rolename="manager-gui"/>
    <role rolename="manager-script"/>
 <user username="admin" password="admin" roles="manager-gui,manager-script"/>
</tomcat-users>

密码要复杂点阿

最后保存，回到Jenkins保存 - 左面列表 > 立即构建 ok了

YApi接口管理平台搭建时的记录

post @ 2018-08-07

YApi接口管理平台搭建时的记录

无意发现了这个接口管理平台，觉得挺不错 开始搭一个吧

一共有两种部署方式 （方式一. 可视化部署、方式二. 命令行部署）

我选择的是命令行部署

首先环境需要 nodeJs、mongoDB

需要注意的是

1.确保 node 版本=> 7.6, 可运行 node -v 查看版本号

2.确保 mongodb 版本 => 2.6，可运行 mongo –version 查看版本号

3.确保安装了 npm, 可运行 npm -v 查看版本号

4.确保安装了 node-gyp 环境，配置方法 运行

npm install --global --production windows-build-tools

这些都可以之后 到Git将包下载下来

https://github.com/YMFE/yapi.git

首先创建一个 “yapi” 文件夹 ，然后 在yapi文件夹下创建 “vendors” 文件夹

将下载下来的包解压，将里面所有的文件复制到 “vendors”文件夹下面

找到解压出的文件中的 “config_example.json” 复制到 “yapi” 文件夹下，并改名为 “config.json” 在”config.json”中可以配置访问端口和访问数据的连接

cmd命令进入到 “E:\yapi\vendors” 下 (我的放在E盘了)

执行命令

npm install --production --registry https://registry.npm.taobao.org

等待完成后 继续执行 npm run install-server 安装程序会初始化数据库索引和管理员账号

最后执行命令 node E:\yapi\vendors\server\app.js 启动服务

打开 127.0.0.1:3000 ok

MongoDB在Windows下安装时遇到的问题

post @ 2018-08-07

MongoDB在Windows下安装时遇到的问题

搭建项目时需要用到MongoDB,之前没接触过

在本机按照runoob.com的教程安装了一遍 挺顺利的,

就是需要注意最后一步时左下角的 “Install MongoDB Compass”

图形化界面如果不取消勾选 要等很久很久很久(电脑开了一晚上)都没安装完

ok 本机没问题 开始服务器搭建，第一步就出现问题了，可能是因为服务器环境的原因，安装时一直提示权限不足 安装不了。

搜了一下 https://www.mongodb.org/dl/win32 官网提供了免安装包 可以对应的下载，配置好环境变量 “Path” 我的放在了E盘符下

在解压好的 “mongoDB” 文件夹下建立好 “date” 与 “log” 文件夹

cmd控制台 命令进入到 “E:\mongoDB\bin”

执行命令 mongod--dbpath E:\mongoDB\data 配置

无意外 打开 localhost:27017 时页面就会给予提示了

然后我想更换端口为8098 查了一下

更换端口的命令 控制台执行命令 

mongod --dbpath E:\MongoDB\data --port 8098

这时提示我有错误了，搜了一下 是因为有文件锁住了的原因

在”mongoDB\data”下有 “mongod.lock” 与 “storage.bson” 这两个文件，将这两个删除后 重新执行就好了

接着 想转成Window服务

首先 先把当前服务关掉, 然后把 “mongoDB\data”下有 “mongod.lock” 与 “storage.bson” 这两个文件删除掉

如果当前已经有Window服务了就需要移除一下

执行移除命令

mongod --logpath "E:\MongoDB\log\MongoDB.log" --logappend --dbpath "E:\Mongodb\data" --directoryperdb --serviceName "MongoDB" --serviceDisplayName "MongoDB" --remove

再继续执行添加服务命令

mongod --logpath "E:\MongoDB\log\MongoDB.log" --logappend --dbpath "E:\MongoDB\data" --port 8098 --directoryperdb --serviceName "MongoDB" --serviceDisplayName "MongoDB" --install

注意 “–port 8098” 要把这个加进去，我搜到的都是只有添加服务，如果不加 “–port 8098” 这个Window服务启动时 服务的端口还是默认的27017，这样就ok了

更改端口后连接数据库后面也要加上端口号才行 (不然还会连默认的27017端口)

mongo --port 8098

微信公众号支付初步开发

用户授权拿到code > 拿code去换access_token > 拿token换用户信息 > 到达输入缴费页面 > 使用JSSDK config接口注入权限验证配置 > 通过ready接口处理成功验证 成功后及可调用其他接口 > 点击支付 > 调用wxPay方法ajax统一下单生成预付订单 > 支付成功回调

项目用JFinal Weixin框架开发的，省了很多事，签名阿之类的，遇到挺多问题，记录下来

首先公众号先要有微信支付功能 申请一下 ,还有你的域名必须是80端口而且备案过的

重要的三个设置

1、设置JS接口安全域名，设置完之后可以在该域名下调用微信开放的JS

2、设置授权回调域名 ，写的时候这个授权域名没设置对 一到用户授权的时候就提示 redirect uri参数错误 调了半天 很难受

3、支付授权的目录， 也就是你发起支付请求 所在页面 ！ 页面！ 页面！的目录 ，我设置的只有域名 结果一直支付失败 只给我 {“errMsg”:”chooseWXPay:fail”} 这就很难受了 只知道失败了 重新检查了一下 设置了一下授权目录
还有api支付的密钥 一定要和自己设置的一样

微信还提供了公众平台测试账号 在里面可以测试其他功能 但是测试不了微信支付
