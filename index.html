mySql分组后的所有数据列根据组内的某字段倒序展示

场景

用户数据在A表

员工操作修改时不允许更新A表数据要单独存另外B表里（存在多次修改），但查询时要显示最新的

那么 写个子查询 在子查询中排序，然后再分组就可以了

SELECT `name`,age 
FROM (
    SELECT a.`name` AS `name`,b.age AS age 
    FROM tb_a a
    LEFT JOIN b ON a.`name` = b.`name`
    ORDER BY b.createTime DESC
    LIMIT 10000000 
 ) dd 
 GROUP BY `name`

要注意的是 mysql 5.7之后子查询内排序是无效的，所以加上limit

Js链式调用思路

post @ 2019-06-26

Js链式调用思路

今天突然想到Js链式调用，效果这样 ↓

> tool.加(1,2,3,4).减(1,1,3).乘(10,10).除(2,2).和();
< 20

代码如下，一个加减乘除的方法

var tool = {
    x : 0,
    加 : (...n) => {
            tool.x += n.reduce((m,c)=>m+c);
            return tool;
    },
    减 : (...n) => {
            tool.x += n.reduce((m,c)=>m-c);
            return tool;
    },
    乘 : (...n) => {
            tool.x *=n.reduce((m,c)=>m*c);
            return tool;
    },
    除 : (...n) => {
            for(var x of n){
                tool.x/=x;
            }
            return tool;
    },
    和 : () => {
            var x = tool.x;
            tool.x = 0;
            return x;
    }
};

思路就是当调用其中1个方法计算结束后 返回函数tool

这样做 

tool.加(1,2,3).减(1,2,3);

就等同于

tool.加(1,2,3);
tool.减(1,2,3);

当然最后要返回值的嘛，把想计算的写完后再调用 和();将函数中存计算值的变量清空再返回

var x = tool.x;
tool.x = 0;
return x;

在segmentfault还看到另外一种链式调用写法

https://segmentfault.com/a/1190000008724608?utm_source=tag-newest

效果

>add(1)(2)(3) //6

代码

function add (num) {
    var count = num;
    var _b = function(l){
        count += l;
        return _b
    }
    _b.valueOf = function(){
        return count
    }
    return _b
}

在方法体内部通过私有方法实现运算，由于闭包 count得以累加 利用valueOf来存返回值，
最后返回计算方法b，而再次调用时调用的是前一次返回的b方法本身，这样就实现了链式。

最后就是得到的结果, b并不是数字而是一个方法，重写valueOf指定了_b方法的原始值，所以console.log输出时_b显示的是计算值，而调用b方法时还会继续进行计算

举个栗子 ↓ 

function test(){
    return 5;
}
test.valueOf = function (){ return 20}
console.log(test); // 20
console.log(test());// 5

这样来看就简单了

so 改成

键要加上双引号 这样就可以了
